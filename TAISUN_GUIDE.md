## 1. TAISUN v2 とは何か

### 一言で言うと

```
「Claude Code を本気で使い倒すための、最強の拡張パック」
```

Claude Code 単体では「優秀なAIアシスタント」ですが、
TAISUN v2 を接続すると**専門家チームが24時間稼働するシステム**になります。

### できることの全体像

| 分野 | できること |
|------|-----------|
| **マーケティング** | LP・VSL・セールスレター・Meta広告・SNS投稿を全自動生成 |
| **リサーチ** | 6〜13のAPIを並列実行し、論文からSNSまで横断調査 |
| **システム開発** | 要件定義→設計→実装→テスト→デプロイを自律実行 |
| **動画制作** | シナリオ→台本→サムネイル→編集まで一貫して自動化 |
| **画像生成** | AI画像・サムネイル・ヘッダー・図解を自動生成 |
| **音声/TTS** | テキストを音声（WAV）に変換、VOICEVOX対応 |
| **SEO** | キーワード抽出・分析・コンテンツ戦略を自動立案 |

## 3. エージェントとは何か・並列処理とは

### エージェントとは

エージェントとは「**特定の専門知識を持ったAIスタッフ**」です。

| エージェント例 | 専門分野 | 何をするか |
|--------------|---------|-----------|
| `architect` | システム設計 | アーキテクチャを設計・レビュー |
| `researcher` | リサーチ | 情報収集・分析・レポート生成 |
| `tdd-guide` | テスト | テスト駆動開発で品質保証 |
| `security-reviewer` | セキュリティ | 脆弱性検出・修正提案 |
| `code-reviewer` | コードレビュー | 可読性・バグ・パフォーマンス改善 |
| `data-analyst` | データ分析 | DB集計・可視化・意思決定支援 |
| `ait42-coordinator` | タスク統括 | 他のエージェントを最適に組み合わせる |

TAISUN v2 には **96体** のエージェントが搭載されています。

### 並列処理とは

**並列処理 = 複数のエージェントが同時に動く**ことです。

```
通常のAI（順番に処理）:
  タスクA → 完了 → タスクB → 完了 → タスクC → 完了
  合計時間: A + B + C

TAISUN v2（並列処理）:
  タスクA ─────────────────→ 完了
  タスクB ─────────────→ 完了     ← 同時に実行
  タスクC ──────────────────→ 完了
  合計時間: max(A, B, C) ← 最も遅いものだけ待てばいい
```

### 実例：競合他社リサーチを依頼した場合

```
ユーザー: 「競合3社を徹底分析して」

TAISUN v2 の動き（すべて同時）:
  Agent 1: A社の公式サイト・SNS・口コミを分析
  Agent 2: B社の価格・機能・戦略を調査
  Agent 3: C社の強み・弱み・顧客評価を調査
  Agent 4: 3社の比較表と差別化ポイントを整理

→ 数分後に完成した比較レポートが届く
```

人間1人が順番に調べると2〜3時間かかる作業が、**数分で完了**します。

### スピード感の目安

| タスク | 通常のAI | TAISUN v2 |
|--------|---------|-----------|
| 競合リサーチ（3社） | 1〜2時間 | 5〜10分 |
| LP作成（12セクション） | 2〜3時間 | 15〜20分 |
| システム要件定義 | 半日〜1日 | 30〜60分 |
| テストコード生成 | 1〜2時間 | 10〜15分 |
| 動画シナリオ作成 | 1〜2時間 | 15〜20分 |

## 4. スキルとは何か

### スキルとは

スキルとは「**実績のある手順書を、いつでも呼び出せる形にしたもの**」です。

```
通常のAI: 毎回ゼロから考える
  → 品質がバラバラ
  → ノウハウが蓄積されない
  → 同じ失敗を繰り返す

TAISUN v2 のスキル: 最適化された手順書を自動適用
  → 毎回同じ高品質な結果
  → 成功パターンが蓄積される
  → 専門家の知恵が凝縮されている
```

### スキルの特徴

| 特徴 | 内容 |
|------|------|
| **再現性** | 誰が呼び出しても同じ品質の結果が出る |
| **専門性** | その分野のベストプラクティスが組み込まれている |
| **拡張性** | 自分のスキルを追加・カスタマイズできる |
| **連携** | スキル同士を組み合わせてパイプライン化できる |

### 呼び出し方

```
/スキル名 と入力するだけ

例:
/mega-research  → 6つのAPIで一気にリサーチ
/taiyo-style-lp → 太陽スタイルのLP作成開始
/sdd-full       → 要件定義から運用手順まで一括生成
```

## 6. MCPツールとは何か

### MCPとは

MCP（Model Context Protocol）は**Claude Code に外部ツールを接続する仕組み**です。

```
MCP なし: Claude Code はAIの頭脳だけ
MCP あり: Claude Code が外部のツール・サービスに直接アクセスできる
```

### TAISUN v2 の MCPツール一覧

| MCPサーバー | できること | 状態 |
|------------|-----------|------|
| **playwright** | 実際のブラウザを自動操作（スクレイピング・フォーム入力・スクリーンショット） | 常時有効 |
| **open-websearch** | Web検索をリアルタイムで実行 | 常時有効 |
| **taisun-proxy** | 内部ツール統合ハブ | 常時有効 |
| **youtube** | YouTube動画の情報取得・コメント分析 | 必要時に有効化 |
| **figma** | Figmaデザインデータの読み取り・書き込み | 必要時に有効化 |
| **qdrant** | ベクトル検索DB（類似文書検索） | 必要時に有効化 |
| **meta-ads** | Meta広告のデータ取得・入稿・最適化 | 必要時に有効化 |
| **twitter-client** | X(Twitter)への投稿・分析 | 必要時に有効化 |
| **pexels** | 商用フリー写真・動画を検索・ダウンロード | 必要時に有効化 |
| **obsidian** | Obsidianのノートを読み書き | 必要時に有効化 |
| **n8n-mcp** | n8nワークフロー自動化との連携 | 必要時に有効化 |
| **sequential-thinking** | 複雑な問題を段階的に思考 | 必要時に有効化 |

### MCPの使い方の例

```
「競合サイトを分析して」と伝えると...

  playwright MCP が起動
    → 実際にサイトをブラウザで開く
    → スクリーンショットを撮る
    → 全テキストを取得
    → JavaScriptの動きを解析

  /url-all スキルと組み合わせ
    → 完全な競合分析レポートが完成
```

## 7. なぜ初めての人でもシステムを作れるのか

### 理由①：「作り方」ではなく「何を作るか」だけ言えばいい

```
従来のシステム開発:
  プログラミング言語を学ぶ
  → フレームワークを学ぶ
  → ライブラリを調べる
  → エラーと格闘する
  → 数ヶ月かかる

TAISUN v2:
  「こんなシステムが欲しい」と伝える
  → /sdd-full で要件定義
  → /build-feature で実装
  → /generate-tests でテスト
  → /deploy でデプロイ
  → 数時間で動くシステムが完成
```

### 理由②：失敗しても自動で修正される

```
通常の開発: エラーが出たら自分で調べて直す
TAISUN v2:
  エラー発生
    → build-error-resolver エージェントが自動起動
    → 原因を分析
    → 最小限の修正を提案・実行
    → 再テスト → 通過
```

### 理由③：13層の防御システムで安全

初心者が「危険なコマンドを実行しそう」「ファイルを消しそう」な場面でも：

```
Layer 1: rm -rf / などの破壊的コマンドを常時ブロック
Layer 2: ベースラインファイルの改変を検出・警告
Layer 3: 未読ファイルの編集を禁止（先にReadさせる）
...
Layer 13: セッション終了時の自動バックアップ
```

### 理由④：「同じ品質」が再現できる

```
人間のエキスパート: その人がいないとできない
TAISUN v2 のスキル: 誰でも呼び出せば同じ品質が出る

/taiyo-style-lp を呼べば → プロのコピーライターが書いたようなLPが出る
/sdd-full を呼べば     → 上級エンジニアが設計したような仕様書が出る
/mega-research を呼べば → 専門リサーチャーが調べたようなレポートが出る
```

### 理由⑤：クロスセッション学習（AGENTS.md）

```
普通のAI: セッションをまたぐと何も覚えていない

TAISUN v2:
  AGENTS.md に教訓が自動蓄積
  → 次のセッションでも前回の失敗・成功パターンを把握
  → どんどん賢くなっていく
```

## 8. インストール・使い方

### インストール（Mac）

```bash
# 1. クローン
git clone https://github.com/taiyousan15/taisun_agent.git ~/taisun_agent
cd ~/taisun_agent

# 2. セットアップ（依存関係 + Hook + スキル + エージェント一括）
npm run taisun:setup

# 3. 動作確認（100点が出ればOK）
npm run taisun:diagnose
```

### 別プロジェクトで使う

```bash
cd ~/your-project
ln -s ~/taisun_agent/.claude .claude
ln -s ~/taisun_agent/.mcp.json .mcp.json
# これだけで全スキル・エージェントが使えるようになる
```

### アップデート

```bash
cd ~/taisun_agent && git pull origin main && npm run taisun:setup && npm run taisun:diagnose
```

### 基本的な使い方

```
# Claude Code を起動
claude

# スキルを呼び出す（/ + スキル名）
/mega-research     → ディープリサーチ開始
/taiyo-style-lp   → LP作成開始
/sdd-full         → 要件定義〜運用手順まで一括生成
/url-all          → URLを徹底分析

# 自然な言葉でも動く
「競合他社3社を徹底的に調べて比較表を作って」
「このシステムの要件定義書を作って、設計して、テストコードも書いて」
「売れるLPを作って。ターゲットは30代の起業家」
```

### 診断コマンド

```bash
npm run taisun:diagnose       # システム全体の健康診断（100点が目標）
npm run taisun:diagnose:fix   # 問題を自動修復
npm run memory:report         # 記憶・学習状況の確認
npm run workflow:start        # ワークフロー開始
```

## AIモデルの自動切替

タスクの複雑さに応じて、コストを最適化しながらモデルが自動切替されます。

| 複雑さ | 使うモデル | 例 |
|--------|----------|-----|
| trivial/simple | **Haiku**（最速・最安） | 挨拶・確認・簡単な検索 |
| moderate/complex | **Sonnet**（バランス型） | ファイル修正・新機能実装 |
| expert | **Opus**（最高性能） | アーキテクチャ設計・セキュリティ監査 |
