# ADR-002: Resilience リトライ強化 + エラー分類

**日付**: 2026-03-14
**ステータス**: accepted
**実装**: `src/proxy-mcp/internal/resilience.ts`

## 背景

エラー率 10-15% の主因の1つが、リトライ戦略の不備。
- `maxAttempts: 2` は少なすぎ（transientエラーは3回目で回復するケースが多い）
- エラー種別を区別せずすべて同じバックオフで再試行 → fatal エラーで無駄なリトライ

## 判断

**エラーを4クラスに分類し、クラス別のリトライ戦略を適用する。**

| クラス | 例 | 戦略 |
|---|---|---|
| `transient` | ネットワーク断、タイムアウト | 指数バックオフ × 5回 |
| `rate_limit` | 429 Too Many Requests | バックオフ係数4倍 × 5回 |
| `logic_error` | 400/422 不正リクエスト | バックオフ係数2倍 × 5回 |
| `fatal` | 401/403 認証エラー | 即座に諦める（0リトライ） |

変更値:
- `maxAttempts`: 2 → 5
- `backoffMs`: 250ms → 500ms

## 期待効果

- エラー率: 15% → 8%（Phase 3単独で）
- fatal エラーの無駄なリトライ排除でレイテンシ改善
