# ADR-001: Output Validation Gate の導入

**日付**: 2026-03-14
**ステータス**: accepted
**実装**: `src/proxy-mcp/validation/output-validator.ts`

## 背景

Omega Research によるシステム分析で以下の問題が判明:
- ハルシネーション率: 20%超（競合: 5%未満）
- エラー率: 10-15%（競合: 5%未満）

出力に不確実な表現（「おそらく」「と思います」等）が含まれていても
そのまま返答されており、品質ゲートが存在しなかった。

## 判断

**3段階バリデーションゲートを出力層に追加する。**

1. **不確実性検出** — 日英26フレーズのパターンマッチ + 重み付けスコア
2. **一貫性チェック** — 過去出力との矛盾パターン検出
3. **スキーマ検証** — 構造化出力の必須フィールド確認

## 検討した代替案

| 案 | 却下理由 |
|---|---|
| LLMによる出力レビュー | コスト高・レイテンシ増大 |
| 人手レビュー | スケールしない |
| なにもしない | ハルシネーション率が競合比4倍のまま |

## 期待効果

- ハルシネーション率: 20% → 10%（Phase 1単独で）
- `shouldRetry: true` で呼び出し元が再生成をトリガー可能

## トレードオフ

- パターンベースのため、検出漏れあり（意図的な「不確か」表現を見逃す可能性）
- スコアリングは휴리スティック — 将来的にMLモデルへ置換予定
